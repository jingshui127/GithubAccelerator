## 2. 总体描述

### 2.1 产品概述

**GitHub 加速器 Pro** 是一款专为解决中国大陆地区 GitHub 访问问题而设计的桌面应用程序。通过智能管理 Hosts 文件，自动选择最优 IP 地址，实现 GitHub 服务的快速稳定访问。

### 2.2 用户特征

| 用户类型 | 描述 | 技术水平 |
|----------|------|----------|
| 普通开发者 | 需要访问 GitHub 的软件开发者 | 中等 |
| 学生用户 | 学习编程的学生群体 | 初级 |
| 企业用户 | 企业开发团队成员 | 中高级 |
| 运维人员 | 负责系统维护的 IT 人员 | 高级 |

### 2.3 运行环境

| 环境 | 要求 |
|------|------|
| 操作系统 | Windows 10 (1803+) / Windows 11 |
| 运行时 | .NET 10 Runtime |
| 内存 | 最低 512MB，推荐 1GB+ |
| 磁盘 | 50MB 可用空间 |
| 网络 | 互联网连接 |

### 2.4 设计约束

- 使用 C# 和 .NET 10 开发
- UI 框架使用 Avalonia 12
- 遵循 MVVM 架构模式
- 支持管理员权限运行（修改 Hosts）

## 3. 功能需求

### 3.1 FR-001 Hosts 数据获取

#### 3.1.1 描述

系统应能够从多个数据源获取最新的 GitHub Hosts 数据。

#### 3.1.2 输入

| 输入项 | 类型 | 说明 |
|--------|------|------|
| 数据源 URL | String | Hosts 数据获取地址 |
| 超时时间 | Integer | 请求超时时间（毫秒） |

#### 3.1.3 输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| Hosts 记录列表 | List<HostRecord> | 解析后的 Hosts 记录 |
| 获取状态 | Enum | 成功/失败/超时 |
| 响应时间 | Long | 请求耗时（毫秒） |

#### 3.1.4 处理流程

```
开始
  │
  ▼
遍历数据源列表
  │
  ▼
发送 HTTP 请求 ──超时──▶ 记录失败，尝试下一数据源
  │
  成功
  ▼
解析 Hosts 数据
  │
  ▼
验证数据有效性
  │
  ▼
返回 Hosts 记录列表
  │
  ▼
结束
```

#### 3.1.5 验收标准

| ID | 标准 |
|----|------|
| FR-001-01 | 支持至少 5 个数据源 |
| FR-001-02 | 单个数据源超时时间可配置（默认 5000ms） |
| FR-001-03 | 自动切换到备用数据源 |
| FR-001-04 | 记录每个数据源的响应时间 |

### 3.2 FR-002 IP 测速优选

#### 3.2.1 描述

系统应能够对获取的 IP 地址进行测速，并选择响应最快的 IP。

#### 3.2.2 输入

| 输入项 | 类型 | 说明 |
|--------|------|------|
| IP 地址列表 | List<String> | 待测速的 IP 地址 |
| 测速端口 | Integer | 测速使用的端口 |
| 并发数 | Integer | 并发测速数量 |

#### 3.2.3 输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 测速结果列表 | List<SpeedResult> | 各 IP 的测速结果 |
| 最优 IP | String | 响应最快的 IP 地址 |
| 排序后的 IP 列表 | List<String> | 按响应时间排序 |

#### 3.2.4 处理流程

```
开始
  │
  ▼
初始化测速引擎
  │
  ▼
并发测速所有 IP
  │
  ├──▶ IP1: Ping/TCP连接测试
  ├──▶ IP2: Ping/TCP连接测试
  └──▶ IPn: Ping/TCP连接测试
  │
  ▼
收集测速结果
  │
  ▼
按响应时间排序
  │
  ▼
标记最优 IP
  │
  ▼
结束
```

#### 3.2.5 验收标准

| ID | 标准 |
|----|------|
| FR-002-01 | 支持 TCP Ping 和 ICMP Ping 两种模式 |
| FR-002-02 | 测速并发数可配置（默认 10） |
| FR-002-03 | 超时 IP 自动标记为不可用 |
| FR-002-04 | 测速结果持久化存储 |

#### 3.3.1 描述

#### 3.3.2 输入

#### 3.3.3 输出

| 输出项 | 类型 | 说明 |

| 错误信息 | String | 失败原因 |

#### 3.3.4 处理流程

```
开始
  │
  ▼

  │

  │
  ▼

  │
  ▼
```

### 3.4 FR-004 数据源性能监控

#### 3.4.1 描述

系统应持续监控各数据源的性能指标，为智能选择提供数据支持。

#### 3.4.2 功能要求

| ID | 要求 |
|----|------|
| FR-004-01 | 记录每个数据源的请求次数 |
| FR-004-02 | 记录成功率和失败率 |
| FR-004-03 | 记录平均响应时间 |
| FR-004-04 | 计算综合健康评分 |
| FR-004-05 | 支持历史数据查询 |

#### 3.4.3 性能指标

```
健康评分 = (成功率 × 40) + (响应时间得分 × 30) + (稳定性得分 × 30)

其中:
- 成功率 = 成功次数 / 总请求次数
- 响应时间得分 = 100 - (平均响应时间 / 最大响应时间 × 100)
- 稳定性得分 = 100 - (连续失败次数 × 10)
```

---

### 3.5 FR-005 智能数据源选择

#### 3.5.1 描述

系统应根据性能数据自动选择最优数据源。

#### 3.5.2 选择策略

```
优先级排序:
1. 健康评分 > 80
2. 最近一次请求成功
3. 平均响应时间 < 500ms
4. 连续失败次数 = 0

选择算法:
IF 存在满足以上所有条件的数据源 THEN
    选择响应时间最短的
ELSE IF 存在健康评分 > 60 的数据源 THEN
    选择评分最高的
ELSE
    选择上次成功的数据源
```

## 6. 数据需求

### 6.1 数据实体

#### 6.1.1 HostRecord

```
HostRecord {
    string Domain;      // 域名
    string IP;          // IP 地址
    string Comment;     // 注释
    DateTime AddedAt;   // 添加时间
}
```

#### 6.1.2 SourcePerformance

```
SourcePerformance {
    string SourceId;           // 数据源 ID
    string SourceName;         // 数据源名称
    string SourceUrl;          // 数据源 URL
    int TotalRequests;         // 总请求次数
    int SuccessCount;          // 成功次数
    int FailureCount;          // 失败次数
    double AverageResponseTime; // 平均响应时间
    double HealthScore;        // 健康评分
    DateTime LastSuccessTime;  // 最后成功时间
    int ConsecutiveFailures;   // 连续失败次数
}
```

#### 6.1.3 OperationLog

```
OperationLog {
    DateTime Timestamp;    // 时间戳
    string Operation;      // 操作类型
    string Details;        // 操作详情
    string Result;         // 操作结果
    string ErrorMessage;   // 错误信息
}
```

### 6.2 数据存储

| 数据类型 | 存储位置 | 格式 |
|----------|----------|------|

| 日志文件 | %AppData%/GithubAccelerator/logs/ | TXT |
| Hosts 备份 | %AppData%/GithubAccelerator/backups/ | TXT |

### 6.3 数据保留策略
