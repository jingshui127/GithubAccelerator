## 1. 产品简介

### 1.1 产品概述

**GitHub 加速器 Pro** 是一款专为解决中国大陆地区 GitHub 访问问题而设计的桌面应用程序。通过智能管理 Hosts 文件，自动选择最优 IP 地址，实现 GitHub 服务的快速稳定访问。

### 1.2 主要功能

| 功能 | 描述 |
|------|------|
| 🚀 一键加速 | 自动获取并应用最优 Hosts |
| 📊 性能监控 | 实时监控数据源性能 |
| 🔄 智能切换 | 自动选择最快数据源 |
| 📈 图表分析 | 可视化性能数据 |
| 💾 数据备份 | 配置和数据安全备份 |
| 🌙 主题切换 | 深色/浅色主题支持 |

### 1.3 适用场景

- 📦 克隆/拉取 GitHub 仓库
- 🌐 浏览 GitHub 页面
- 📥 下载 Release 文件
- 🔍 搜索代码和项目
- 📝 编辑 Wiki 和 Issues

## 3. 安装指南

### 3.1 下载安装

1. 从 [GitHub Releases](https://github.com/jingshui127/GithubAccelerator/releases) 下载最新版本
2. 运行 `GithubAccelerator.Setup.exe`
3. 按照安装向导完成安装
4. 首次运行时，可能需要以**管理员身份**运行

### 3.2 便携版安装

1. 下载 `GithubAccelerator.Portable.zip`
2. 解压到任意目录
3. 运行 `GithubAccelerator.UI.exe`

### 3.3 卸载

- **安装版**: 通过"设置 > 应用 > 已安装的应用"卸载
- **便携版**: 直接删除文件夹

## 4. 快速入门

### 4.1 首次使用

#### 步骤 1: 启动应用

双击桌面图标或开始菜单中的"GitHub 加速器 Pro"启动应用。

#### 步骤 2: 授予权限

首次使用时，系统会提示需要**管理员权限**来修改 Hosts 文件。点击"是"授权。

#### 步骤 3: 刷新数据

应用启动后会自动刷新数据源，也可以点击"🔄 刷新"按钮手动刷新。

#### 步骤 4: 应用加速

1. 查看数据源列表，选择响应时间最短的源
2. 点击"✅ 应用选中源"按钮
3. 等待提示"应用成功"

#### 步骤 5: 验证效果

打开浏览器访问 [github.com](https://github.com)，验证访问速度是否改善。

## 5. 功能详解

### 5.1 仪表盘

仪表盘是应用的主界面，显示核心监控指标和操作入口。

#### 5.1.1 监控指标卡片

| 卡片 | 说明 |
|------|------|
| 监控状态 | 显示当前是否在监控中 |
| 最佳响应 | 所有数据源中最快的响应时间 |
| 综合评分 | 最优数据源的健康评分 |
| 数据源 | 可用数据源的总数量 |

#### 5.1.2 快速操作栏

| 按钮 | 功能 | 快捷键 |
|------|------|--------|
| ▶ 开始/⏹ 停止 | 开始或停止监控 | F5 |
| 🔄 刷新 | 重新获取数据源 | Ctrl+R |
| ✅ 应用选中源 | 应用勾选的数据源 | Ctrl+S |
| 🗑️ 清除Hosts | 清除已应用的加速记录 | - |

#### 5.1.3 推荐数据源

系统根据健康评分自动推荐最优数据源，点击"应用"按钮即可一键加速。

#### 5.1.4 数据源列表

显示所有可用数据源及其性能指标：

| 列 | 说明 |
|------|------|
| 复选框 | 勾选要应用的数据源 |
| 状态指示灯 | 绿色=正常，黄色=较慢，红色=异常 |
| 名称 | 数据源名称 |
| URL | 数据源地址 |
| 响应时间 | 最近一次请求的响应时间 |
| 评分 | 综合健康评分 (0-100) |

---

### 5.2 性能图表

性能图表页面提供可视化的性能数据分析。

#### 5.2.1 响应时间趋势图

显示最近一段时间内各数据源的响应时间变化趋势。

- **X轴**: 时间
- **Y轴**: 响应时间 (ms)
- **曲线**: 不同颜色代表不同数据源

#### 5.2.2 数据源对比图

以柱状图形式对比各数据源的平均响应时间。

#### 5.2.3 评分分布图

显示各数据源健康评分的分布情况。

---

### 5.3 智能数据源管理

智能数据源管理功能帮助用户自动管理和选择最优数据源。

#### 5.3.1 功能特点

| 功能 | 说明 |
|------|------|
| 自动健康检测 | 启动1小时后自动开始定期检测数据源 |
| 每小时检测 | 每小时自动检测所有数据源的可用性 |
| 智能选择 | 自动选择响应最快的可用数据源 |
| 状态显示 | 在仪表盘显示可用数据源数量 |

#### 5.3.2 工作流程

1. 应用启动后运行1小时开始首次检测
2. 每小时自动检测所有数据源
3. 记录各数据源的响应时间和成功率
4. 自动跳过不可用的数据源
5. 优先使用稳定快速的数据源

#### 5.3.3 查看数据源状态

在仪表盘的数据源卡片中可以查看：
- 总数据源数量
- 可用数据源数量

## 6. 设置配置

### 6.1 配置文件位置

配置文件存储在以下位置：

```
%AppData%\GithubAccelerator\
├── config.json          # 应用配置
├── data\                # 数据目录
│   └── performance.json # 性能数据
└── backups\             # 备份目录
```

### 6.2 配置文件格式

`config.json` 示例：

```json
{
  "version": "1.1.0",
  "autoStart": true,
  "startMinimized": false,
  "autoMonitor": true,
  "refreshInterval": 300,
  "timeout": 5000,
  "isDarkMode": true,
  "language": "zh-CN"
}
```

### 6.3 重置配置

如需重置所有配置：

1. 关闭应用程序
2. 删除 `%AppData%\GithubAccelerator\config.json`
3. 重新启动应用程序

## 7. 常见问题

### Q1: 应用后仍然无法访问 GitHub？

**解决方案**:
1. 检查是否以管理员身份运行
2. 清除 DNS 缓存：打开 CMD，执行 `ipconfig /flushdns`
3. 尝试刷新数据源后重新应用
4. 检查是否有其他代理软件冲突

### Q2: 提示"需要管理员权限"？

**解决方案**:
1. 右键点击应用图标
2. 选择"以管理员身份运行"
3. 或在属性中设置"以管理员身份运行此程序"

### Q3: 数据源全部超时？

**解决方案**:
1. 检查网络连接是否正常
2. 检查防火墙是否阻止应用
3. 等待一段时间后重试
4. 尝试使用代理

### Q4: 应用后访问变慢了？

**解决方案**:
1. 点击刷新按钮重新测速
2. 选择响应时间更短的数据源
3. 清除 Hosts 后重新应用

### Q5: 如何完全卸载？

**解决方案**:
1. 通过系统设置卸载应用
2. 手动删除 `%AppData%\GithubAccelerator` 文件夹
3. 检查并清理注册表自启动项

### Q6: 开机自启不生效？

**解决方案**:
1. 检查设置中是否勾选了"开机自启"
2. 检查是否被安全软件拦截
3. 手动添加到启动文件夹
